Kinstrife Database Basics

When to use it

Static data Read-only Multiple instances of an ‘archetype’ Be creative! Many things can become more flexible when based on database entries

How to use it

Access entries

Get the DatabaseAccessor and use its methods TryGetEntry and/or TryGetEntriesOfType

Create and edit entries

  1. Open the database editor window (located at the top bar via Kinstrife -> Database) and navigate to your new type via the type picker at the top left.
  2. The menu on the left will now show all instances of that type it can find on the left. You can either select one and edit its values, or create a new one (which is the bottom most option.)
  3. Clicking the 'Add New' option will create a new preview instance which you can modify. But you will see a warning that it is not yet created - to do so, enter a path at the bottom of the window. The path must be in a Resources/Database/ folder. You can enter either a name for a directory, or a file name (which will need to end with '.asset'). Once you've done so, a Create button will appear, which creates an instance at the path upon clicking. If it doesn't, make sure to check the console for errors. Hint: you can copy a path by selecting a folder in the project view and hitting ctrl+alt+c.
  4. Once you created the type, it will show up in the project view in the path you specified and will also be modifiable via the database editor.

Create database entry types

Create an new database type script via "Create/Kinstrife/Script Templates/Database Type". In this script, you must make sure that the data is immutable during runtime (but still editable in the editor.) A good way to do this is to create private backing fields decorated with the [SerializeField] attribute and allow public access via a get-only property. Because this is a lot of boilerplate writing, this package contains three visual studio code snippets, PropertyWithBackingField and PropertyListWithBackingField. Read the section ‘Add a code snippet to visual studio’ on how to install snippets if you don’t know yet. To use them, type src or srf, respectively, and hit tab twice. You can then edit the type and name of the field & property.

Remember to update the database creation menu in your newly created script file!

What it is

The database is essentially a giant Dictionary<GUID, ScriptableObject>. The DatabaseAccessor allows convenient retrieval of elements by GUID and type. The DatabaseLocator locates potential folders that should get injected to the database (right now everything in Resources/Database) by the DatabaseInjector.
