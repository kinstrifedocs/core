ConfigRegistry

Loads and initializes ConfigData instances from Resources/Config folders. Also binds these as singletons in the DI system. In builds, it exports the default config data (from the resources folder) to the streaming assets folder as JSON, so that it may be modified by players. It then regularly checks the streaming assets folder for modifications to these exported files and serializes them back. When the config registry is first initialized, it also checks the streaming assets folder for modified config data.
