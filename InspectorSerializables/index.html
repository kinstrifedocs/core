Inspector Serializables

"com.kinstrife.core.inspector-serializables": "file:/GameDev/Core/Packages/InspectorSerializables"

 

Adds common types that are not serialized by Unity out of the box, such as dictionaries as well as type and interface references. These can now be serialized and displayed in the editor (via workarounds).

 

Serializable dictionary

Inherit from SerializableDictionary<TKey, TValue> (from the Kinstrife.Core.InspectorSerializables namespace) with the concrete types you want to use, e.g. IntStringDictionary : SerializableDictionary<int, string>. The newly created class must be decorated with the [System.Serializable]attribute. The concrete subclass can then be serialized as expected.

 

Serializable interfaces

Based on the IUnified asset. If there are tricky bugs, try checking if there has been an update on the asset store and/or try contacting the asset author.

Inherit from IUnifiedContainer<TInterface> to create a concrete subclass for the interface you want to reference. The newly created class must be decorated with the [System.Serializable]attribute. The concrete subclass can then be serialized as expected. The interface reference can be retrieved and set via the Result property. Keep in mind that when using this in the editor, this only works for referencing classes that 1) implement the interface and 2) inherit from UnityEngine.Object, other object references cannot be serialized by Unity.

For collections: When iterating over collections of IUnifiedContainer subclasses, you can access values directly as interfaces instead of using the Result property, for example:

interface ISomeInterface { void DoX(); }

[System.Serializable]
class SomeInterfaceContainer : IUnifiedContainer<ISomeInterface> { }

List<SomeInterfaceContainer> myInterfaceList = new List<SomeInterfaceContainer>();

void Test()
{
    foreach (ISomeInterface someInterface in myInterfaceList)
        someInterface.DoX();
}
