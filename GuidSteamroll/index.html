GUID/Database reference steamrolling

"com.kinstrife.core.guid-steamroll": "file:/GameDev/Core/Packages/GuidSteamroll"

When saving the game state, database entries should not be serialized. Instead, references to database entries should be stored. This is possible by referencing database entries through their GUID. However, constantly handling GUIDs in gameplay logic is rather inconvenient, because it requires data consumers/readers to look up the reference in the database with each use.

Instead, all database entries should be accessed directly and should only get 'reduced' to GUIDs for saving. And it should happen 'automagically' for each type, of course, without additional boilerplate code. Both of this is achieved through 'steamrolling'.

Steamrolling works by decorating database entry references (in dynamic data classes) with an attribute and supplying a corresponding (serialized) GUID field or collection that the database entry reference is flattened into.
