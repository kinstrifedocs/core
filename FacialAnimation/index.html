Actor

com.kinstrife.core.actor": "file:/GameDev/Core/Packages/Actor"

Package containing technical functionality with which actors can be defined and preprocessed for authoring. The following functionality & components are available:

Actor

Base data container component containing reference to the Skeleton & Animator. Used as primary access point for referencing an actor.

HumanActor type is available for actors that are human

Skeleton Definition

Asset in which the result from the skeleton extraction is stored as the ground truth for the avatar. Used for retargeting purposes. Contains a mapping of <string, BoneAsset>.

If the skeleton is human, a HumanSkeletonDefinition ought to be used, which contains a reference to the Unity Avatar

BoneAsset

Asset that stores the state for an individual bone in BoneInfo. Can be referenced by other functionality to refer to a specific bone.

BoneInfo

Stores transform matrix (TRS) & name for a specific bone. Equivalent to Unity's SkeletonBone, but serialized for storage.

Skeleton

Component for extracting a skeleton definition in bind pose from a transform hierarchy. It contains a mapping of <string, BoneTransform> which can be used to retrieve bone transforms when e.g. transferring skinned meshes between actors. Also used as a runtime definition for the actor's skeleton after modifications are made to it.

BoneTransform

Instanced / runtime equivalent wrapper of BoneInfo, also containing a reference to its Transform in the hierarchy.
