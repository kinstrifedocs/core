Identification and GUID

"com.kinstrife.core.identification" : "file:/GameDev/Core/Packages/Identification"

 

Motivation

When serializing data (e.g. for storing it on a disk or sending it over a network) it can be convenient to be able to reference data objects indirectly via an identifier. This is the case, for example, with database entries, which should not be saved directly, but instead should only have some sort of reference serialized into a savegame.

 

GUID

C# has a very nice built-in identifier: the GUID struct. Unfortunately, Unity cannot serialize this struct and it cannot be nicely included/saved in e.g. ScriptableObjects. As a result, this package contains a custom GUID type which contains a regular System.Guid as well as a version that is serialized as a string (which can be serialized by Unity's serializer).

 

Built-ins

For convenience's sake, there are some pre-made identifiable classes:

  • IdentifiableObject, a standard ScriptableObject except with a Kinstrife Guid
  • IdentifiableBehaviour, just as above, except as a MonoBehaviour
  • IIdentifiable, requiring the type to publically return a System.Guid
