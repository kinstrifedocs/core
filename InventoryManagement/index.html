Inventory Basics

"com.kinstrife.core.inventory-management": "file:/GameDev/Core/Packages/InventoryManagement"

 

This package handles boilerplate code for managing 'belongs to'-type relationships and transactions.

Intro

In games, objects often have a 'belongs to'-type relationship: Player inventories, equipment, assigning functions to buildings etc. Similarly, many features in games can be broken down to transactions. Equipping items, buying from merchants, transferring people from one group to another can be described as the same action.

Quickstart

This package contains generic abstract classes as templates - they are yours for the taking! In many cases, you only need to choose your inventory types and add a Dictionary field, perhaps with a MessagePack attribute for serialization. You can of course add further additional logic - but you don't have to Ü

Currently implemented templates are:

  • UnlimitedInventory: For when you just need to keep track of numbers
  • LimitedInventory: Useful if you have something like a maximum capacity or other logic that limits whether something can be added
  • SlotInventory: A classic for equippable items, but can have other versatile uses

Interfaces

Functionality of this package is defined via interfaces. While in most cases inheriting from the templates above should be enough, you can simply implement the interface to ensure interoperability with all other inventories.
