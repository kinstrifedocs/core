Dependency Injection (DI) basics

"com.kinstrife.core.dependency-injection": "file:/GameDev/Core/Packages/Dependency Injection"

Intro

This package contains a lightweight, performant dependency injection framework. It can be used to turn your application into a collection of loosely-coupled parts with highly segmented responsibilities.

Quickstart

Dependency Injection requires a Context , a InjectableSourcer, and a controller to direct the application flow. If the scene exists outside of an application flow environment, the standalone variant of the InjectableSourcer is used (DIInitializer).

  1. Create a GameObject with a Context
  2. Add a InjectableSourcer
  3. Add a controller that runs the injection (DIInitializer or custom, e.g. Module)

Core Concepts

Context

The Context exists at the core of the dependency injection functionality, its primary function is to reference the Container, and run the installation.

Installer

Installers are run on the Context as a way to bind dependencies to a container. They are available as either a Monobehaviour or ScriptableObject.
